spotifypi

  • spotifypi is an application I built for the Raspberry Pi to stream music on my stereo and control it with any device in my local network. The name is a composition of the words Spotify and Raspberry Pi.

    When I ordered my Raspberry Pi, which was the very first model on the market, I was very excited about it. But as soon as I had it at home, I did not really have any great ideas what to do with it. One day I thought, it would be awesome to be able to stream music from Spotify directly to my stereo without plugging my smartphone in. After some googling around I did not find any application that did everything I wanted it to do, so I decided to implement it by myself.

    Spotify provided an API that allows you to search for music, access the meta data and stream it. The API could be accessed in C, so most of spotifypi is written in C.

    The most important part for me was the easy access from any device. My initial thought was to build an app for my smartphone to connect with the Raspberry Pi. But because we have Android-, iOS- and Windows devices in our home, I decided to connect to the Raspberry Pi via one very common interface which is installed on every relevant device: the web browser.

    So when the Raspberry Pi boots up, it starts an Apache HTTP Server running PHP, and it starts the C-program I wrote that accesses the Spotify API. When I navigate to the IP address of the Raspberry Pi, I get the website that you can see in the picture below. There you can search for songs or artists, play and pause songs, navigate forwards and backwards and toggle shuffle mode.

    What happens when I pick a song on my smartphone? – The browser performs an AJAX call to the REST API provided by the server. The server communicates with the C-program, which accesses the Spotify API, streams the song you picked and plays it through the sound output of the Raspberry Pi.

    I used this system almost every day, until Spotify decided to discontinue their API, so unfortunately this program is deprecated today.

2048

  • 2048 is a well-known game which is implemented on various websites and apps. The goal is, to slide the tiles until you get to a tile with the value 2048 on it. When you slide horizontally or vertically, tiles with the same value on it will add up to one tile.

    Initially that game was a project for the course Modern Programming Languages in university. The game is written in Scala, which is a functional programming language that compiles into bytecode for the Java Virtual Machine. For the server side it uses the Play Framework which is a very modern and easy-to-use framework for Scala or Java, that allows you to set up a HTTP server with a nice REST API.

    Under the hood, there is a strict Model-View-Controller (MVC) architecture. With that architecture, it is very easy to run multiple user interfaces in parallel. So when starting up the game, it opens a web UI in the web browser, a graphical desktop UI implemented with ScalaFX and a textual UI in the server's console. Moves can be performed in all of the UIs, its results will be propagated to the other UIs.

    The game and its web UI is deployed on Heroku, so have a look at play-2048.herokuapp.com!
